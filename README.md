# Dify--trend-crawler
本项目是一个基于 Python 爬虫与 Dify 工作流构建的自动化舆情监测工具。通过封装定制化的数据采集接口，并结合大模型的 Web 搜索与文本分析能力，实现对目标消费趋势与早期舆情的精准感知。

本工具的运行逻辑分为三个核心阶段：

1. 数据采集与接口封装

为解决动态网页的数据获取难题，底层采用 Python 结合 DrissionPage 工具应对基础反爬限制。这里使用的是Nemo2011大佬的API，十分感谢：
https://github.com/Nemo2011/bilibili-api

操作逻辑：先根据指引部署好自己的cookie，通过监听视频平台（目前为b站，后续会继续跟进）的网络请求接口，自动化提取视频弹幕与核心评论内容（单次抓取量可达 900+ 条）。

服务化：将上述核心采集逻辑封装为本地的 FastAPI 后端接口，提供标准的 HTTP 请求端点，便于上层工具调用。

2. 外部信息检索与语料扩充

为了解决单一社交平台数据维度受限的问题，系统引入了外部网页的交叉检索。

操作逻辑：在 Dify 工作流中直接调用并配置 Dify 自带的 Web 搜索工具并进行迭代搜索，确保信息可靠全面。

功能目标：针对特定的新兴消费趋势或突发事件，Agent 会自动抓取搜索引擎的最新结果及外部网页文本，作为背景信息的强力补充。

3. 大模型工作流编排与报告生成

最终的数据清洗与研判在 Dify 平台完成闭环。

操作逻辑：在 Dify 画布中搭建 LLM 分析工作流。将 FastAPI 接口抓取的“社交语料”与 Web 搜索工具获取的“网页文本”共同接入工作流。

功能目标：设置专属的 LLM 节点对多源数据进行清洗、去除冗余噪音并执行情绪倾向分析。最终将零散的非结构化文本转化为结构化数据，输出针对特定目标的趋势分析与早期舆情报告。
