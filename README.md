# Dify--trend-crawler
本项目是基于Dify平台构筑的网络舆情分析和信息搜索整合项目，可对热点事件进行快速了解解读
通过 FastAPI 引擎，为 LLM 工作流提供从“高价值视频初筛”到“实时弹幕与深度评论深挖”的全链路结构化数据支撑，适用于量化投研、新兴消费趋势监测与危机舆情预警等场景。
🌟 核心特性微服务解耦架构：
将“全局视频检索”与“单视频深度挖掘”拆分为独立服务，支持不同频次和业务场景的按需调用。开箱即用的大模型挂载：内置符合 OpenAPI 3.1.0 规范的 yaml 描述文件，支持零代码无缝接入 Dify 等工作流平台。精细化数据调控：支持大模型基于上下文，动态决策弹幕抓取数量（情绪样本）与评论挖掘深度（观点样本）。隐私与工程规范：配置统一的 .env 鉴权管理与全局异常捕获，无硬编码隐私风险。

🚀 快速开始
1. 环境准备确保你的本地环境已安装 Python 3.8+。克隆本项目后，安装必需的依赖库：pip install fastapi uvicorn pydantic bilibili-api-python
2. 凭证配置本项目依赖真实的 B 站 Cookie 凭证来获取高阶数据。复制根目录下的 .env.example 文件，并重命名为 .env。在浏览器中登录 B 站，获取并填入你的 SESSDATA、BILI_JCT 和 BUVID3。3. 启动微服务本项目包含两个独立的服务端点，请根据需求在独立的终端中启动：
服务 A：视频详情与情绪深挖 API (默认端口 8002)python bilibili_details_api.py
服务 B：投研雷达与初筛搜索 API (默认端口 8003)python b站关键词搜索.py
3. Dify 平台接入进入 Dify 的“工具”面板，选择“自定义工具”。导入本项目提供的 openapi.yaml 和 search_openapi.yaml 文件。验证端点可用性后，即可在你的智能 Agent 工作流中直接拖拽使用。
